## 前四週心得與解題心得

### 第一週 ---- 筆記心得是天空，走過才是踏實

先接觸到的是 terminal 跟 git ，很努力寫了有趣的整理筆記，雖然頭腦好像很清楚，不過沒有自己體驗過，實在不夠踏實，交完作業後，還是要再一直回去看筆記，下個步驟該怎麼做，幾次來回後才能夠熟練。所以筆記心得是要寫的，不過還是要自己操作過才會讓大腦跟身體記起來，如果操作一次不夠，那就兩次，直到記起來為止。

### 第二週 ---- 變數與函式的名稱

之前因為只有自己一個人，看著 Lidemy 上面的課程自己練，沒有人可以糾正我，導致變數命名的亂七八糟，都是 a b c ，然後如果一直寫不出來，還會命名 gg 之類的，現在想來也是滿有趣當，當時的想法是我就知道那是什麼阿，就只是一個名稱，越短越好，這樣打起來比較快，但是現在回頭看程式碼，變得要重頭看，重新理解題目，這樣曠日廢時。

關於變數命名的重要性，有一位助教說，如果變數命名的好，可以讓看 code 的人，甚至不用看你這個程式是要處理什麼問題，就可以知道在做什麼，所以要朝這個目標前進。

函式命名都是用動詞開頭，因為這個函式是要做一件事情。

這週的挑戰題大多都有完成，還是能力範圍內，大數乘法最後寫出來時也是很開心，因為是用迴圈處理，加上有進位的問題，更清楚了迴圈，感謝幫我改第二週作業的助教，不厭其煩的協助調整作業。

### 第三週 ---- eslint 地獄

像是上面所說，之前自己亂搞，導致我的程式碼不堪入目，這時遇到了 eslint 糾察隊，真的快要被搞瘋了，error 一次都是 20、30個跳出來的，開始從分辨 space tab，符號前後都要空格，花括前也要空格，總之就是一直打空格😅
還有被強迫熟悉箭頭函式，之前只覺得是個酷炫的東西，反正我是用正常寫法也可以，不過看了看說是因為程式碼的簡潔所以要用，之後的程式碼一定都是上百行的，所以簡潔好讀是重要的。

這週的挑戰題我覺得真的很挑戰，所以就跳過了。

### 第四週 ---- eslint 地獄 ＋ 串 API

因為這裡的程式碼更多了，其中有一份作業 eslint error 還多達 90 幾，我覺得花在 eslint 上面的時間超多的。

學期到了使用 switch case ，一開始我也覺得沒有必要，但是用 switch case 寫好後，變數與函式命名好後，真的一目了然，實在很神奇。
印象深刻的是 hw3 ，第一次做的時候還弄成處理字串的方式，不知道可以用 . 去取得 object 裡面的值，重寫完之後也滿佩服之前的自己。😂
